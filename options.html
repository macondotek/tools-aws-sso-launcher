Configuration Guide

Organizations Configuration

The Organizations section is used to define the defaults used by each defined account.

To configure an organization, you enter the default region, role name, SSO base URL, which will be used to create the links to the console.

You can also define alternative roles that will be shown on the edit form of the account link to login using an alternative role.

[OrganizationName] region = us-east-1 roleName = Administrator baseURL = https://d-XXXXXX.awsapps.com altRoles = Developer, PowerUser default = true [AnotherOrg] region = us-west-2 roleName = Developer baseURL = https://d-123456.awsapps.com

Accounts Configuration

The Accounts section is used to define the accounts that will be shown in the popup.

To configure an account, you enter the account ID, the organization it belongs to, and an additional group configuration is also available to group the accounts in the popup.

You can also override the default region and role name for the account.

[AccountName] aws_account_id = 123456789012 defaults = OrganizationName group = GroupName [AnotherAccount] aws_account_id = 123456789012 defaults = OrganizationName group = GroupName region = us-west-2 #override the default region roleName = Developer #override the default role name [AnotherAccount2] aws_account_id = 123456789012 defaults = OrganizationName group = Group2 #this will be shown in the popup as a separate group
